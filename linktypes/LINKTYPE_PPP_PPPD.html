LINKTYPE_PPP_PPPD

Description

PPP frames, as specified by Section 2 of RFC1661.

The frame is preceded by a header that is similar to the header for PPP in HDLC-like framing, as specified by Section 3.1 of RFC1662. However, the first octet of the header is not the address field; instead, it is 0x00 if the frame was received by the host that captured the traffic and 0x01 if the frame was sent by that host. The frame includs no flag octets, with the PPP header following the direction and control fields. The data in the frame is not octet-stuffed or bit-stuffed.
